# Manifold Test Suite

This directory bridges the [MeshBool](https://github.com/JaminKoke/meshbool)
Rust crate to the upstream
[Manifold](https://github.com/BorgerLand/manifold/tree/meshbool-test) C++ test
suite. The C++ wrapper headers translate the Manifold API surface to calls into
MeshBool via [Zngur](https://github.com/HKalbasi/zngur)-generated FFI bindings.

> **Note:** These bindings are for testing only and are not optimised for
> general use. They are not the main focus of the project.

---

## Repository layout

```
meshbool/                  ← repo root (Rust crate)
├── Cargo.toml
├── src/                   ← Rust source (do not modify)
│   └── zngur_stubs.rs     ← thin stubs required by the zngur binding layer
├── test/                  ← everything in this directory
│   ├── test.zng           ← zngur interface declaration
│   ├── generated/         ← generated by `cargo build --features test` (gitignored)
│   │   └── meshbool/
│   │       ├── meshbool.h
│   │       └── generated.cpp
│   ├── manifold/          ← git submodule: BorgerLand/manifold@meshbool-test
│   │   ├── CMakeLists.txt ← entry point for cmake (point cmake here)
│   │   ├── include/
│   │   │   └── manifold/  ← wrapper headers (replace upstream originals)
│   │   ├── src/
│   │   │   ├── meshIO.cpp        ← OBJ read/write (extracted from upstream)
│   │   │   ├── cross_section.cpp ← Clipper2-based 2D ops (restored from upstream)
│   │   │   ├── tri_dist.h        ← triangle distance (kept from upstream)
│   │   │   └── utils.h           ← minimal test helpers (kept from upstream)
│   │   ├── test/          ← upstream test files (with some tests #if 0'd out)
│   │   └── samples/       ← upstream sample geometry
│   └── build/             ← cmake build directory (gitignored)
```

## Prerequisites

| Tool | Version |
|------|---------|
| Rust | stable (1.80+) |
| CMake | 3.23+ |
| C++ compiler | C++17, e.g. GCC 11+ or Clang 14+ |

Clipper2 and GoogleTest are fetched automatically by CMake.

## Building

### 1. Clone with submodules

```bash
git clone --recurse-submodules https://github.com/JaminKoke/meshbool.git
cd meshbool

# or, after a plain clone:
git submodule update --init --recursive
```

Make sure the submodule is on the `meshbool-test` branch:

```bash
cd test/manifold
git checkout meshbool-test
cd ../..
```

### 2. Generate the zngur bindings (Rust step)

This compiles the Rust crate with the `test` feature, which runs the `build.rs`
script to invoke `zngur` on `test/test.zng` and write the generated C++/Rust
glue into `test/generated/`.

```bash
cargo build --features test
```

The generated files (`test/generated/meshbool/meshbool.h` and
`test/generated/meshbool/generated.cpp`) are gitignored and **must** be
regenerated whenever `test/test.zng` changes.

### 3. Configure and build the C++ test binary

```bash
mkdir test/build && cd test/build

cmake ../manifold \
  -DMANIFOLD_TEST=ON \
  -DMANIFOLD_CROSS_SECTION=ON \
  -DCMAKE_BUILD_TYPE=Debug

make manifold_test -j$(nproc)
```

> **Why `cmake ../manifold`?**  The `CMakeLists.txt` lives directly in the
> `test/manifold/` submodule; there is no symlink at `test/CMakeLists.txt`.
> Point cmake at the submodule root.

The test binary is produced at:

```
test/build/test/manifold_test
```

### 4. Run the tests

```bash
cd test/build
./test/manifold_test
# or via ctest:
ctest --output-on-failure
```

## How the binding layer works

```
test/test.zng
    │  declares the Rust types/methods exposed to C++
    ▼
cargo build --features test
    │  runs zngur, writes:
    ├─ test/generated/meshbool/meshbool.h     (C++ declarations)
    └─ test/generated/meshbool/generated.cpp  (C++ ↔ Rust glue)

cmake ../manifold
    │  compiles generated.cpp + wrapper headers + test sources,
    │  links against libmeshbool.a (built by Corrosion)
    ▼
test/build/test/manifold_test
```

The wrapper headers in `test/manifold/include/manifold/` replace the upstream
Manifold headers. They translate the C++ `manifold::Manifold` API into calls on
the `rust::crate::MeshBool` type exposed through `meshbool.h`.

`src/zngur_stubs.rs` adds the thin Rust-side glue that zngur needs but that
doesn't belong in the main `src/` (coordinate accessors for nalgebra types,
`MeshBoolError::is_*` methods, and stubs for unimplemented features like
`refine` and `decompose`).

## Disabled tests

Some upstream tests exercise features not yet implemented in MeshBool and are
wrapped in `#if 0` in the test files:

| Feature | Tests disabled |
|---------|---------------|
| `SmoothByNormals` | `NormalsSmoothByNormalsNoArg`, `GetNormalLegacyContract`, `ErrorPropagationSmoothByNormals`, and smooth_test.cpp tests |
| `MinkowskiSum` / `MinkowskiDifference` | `ErrorPropagationMinkowski` + boolean_test.cpp Minkowski tests |
| `Manifold::ReadOBJ` / `WriteOBJ` (member) | `ObjRoundTrip` |
| `ExecutionContext` / `WithContext` | all of context_test.cpp, `DeepChainDoesNotOverflowNumLeaves` |
| `RayCast` | all RayCast tests in measurement_test.cpp |
| `InvalidTangents` error code | `InvalidTangents` in smooth_test.cpp |

The free-function `ReadOBJ` / `WriteOBJ` (used by `ReadTestMeshGL64OBJ` /
`WriteTestOBJ`) **are** implemented via `test/manifold/src/meshIO.cpp`.
